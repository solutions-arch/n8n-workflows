# Auto-Create Point of Contact (POC) Records
**[AT] Create POC**

> New contacts are added once and shared across all systems instantly.

**Status:** Active | **Systems:** Bubble, Airtable

---

## Before (Manual Process)

1. A new point of contact is identified for a client.
1. Someone manually checks if they already exist in the Bubble portal.
1. If not, they create the record by hand — name, email, role, etc.
1. They then link the POC to the correct client requirement.
1. Duplicates are common because there's no automatic check.

---

## After (Automated)

When a new POC is added, the system automatically checks for duplicates, creates the record if needed, and links it to the right requirement.

**Step 1: A new POC is registered**
A process engineer adds a new contact in Airtable.

**Step 2: Duplicate check runs automatically**
The system searches the portal to see if this person already exists.

**Step 3: New record is created if needed**
Only if the POC doesn't exist yet, a new record is created — no duplicates.

**Step 4: POC is linked to the client requirement**
The contact is automatically connected to the correct project.

---

## Impact

- **No duplicate contacts** — the system checks before creating.
- **Automatic linking** — POCs are connected to the right requirement without manual effort.
- **~3-5 minutes saved per POC** — eliminates the check-and-create cycle.
