# Auto-Create Client Requirements
**[AT] Create PDEV Requirement**

> New staffing requirements appear in the internal portal the moment they're logged.

**Status:** Active | **Systems:** Airtable, Bubble

---

## Before (Manual Process)

1. A process engineer logs a new client requirement in Airtable.
1. They then have to manually open the Bubble internal portal.
1. They look up the correct project and client IDs.
1. They re-enter the same information to create a matching record in Bubble.
1. If they forget or enter the wrong ID, the portal shows stale or incorrect data.

---

## After (Automated)

When a new PDEV requirement is created in Airtable, the internal portal is updated automatically — no copy-pasting, no mismatches.

**Step 1: A new requirement is logged in Airtable**
The process engineer fills out the requirement details as usual.

**Step 2: The system looks up the project and client**
It automatically finds the matching records — no manual ID lookup needed.

**Step 3: A client requirement is created in the portal**
The Bubble internal portal instantly reflects the new requirement.

**Step 4: The record IDs are linked**
Both systems are connected so future updates stay in sync.

---

## Impact

- **Instant portal updates** — clients see new requirements without waiting for manual entry.
- **No ID mismatch errors** — the system handles the cross-referencing automatically.
- **~5 minutes saved per requirement** — eliminates the manual lookup-and-retype step.
