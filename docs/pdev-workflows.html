Automated RFP Processing Pipeline

Active95 nodes

[AT] New RFP Form submission

Before

  • An RFP arrives via email or form
  • Someone manually extracts company, client, POC, and role details
  • They check Airtable one-by-one for existing records
  • New records created; existing ones linked — by hand
  • For multi-role RFPs, the entire cycle repeats per role
  • Attachments downloaded and uploaded manually
  • Confirmation emails sent manually
  • Total: 30–60 minutes per RFP

After

  • RFP form submitted once
  • Company, client, POC deduplicated automatically
  • Multi-role RFPs split — each role gets its own requirement
  • Attachments fetched and stored to correct records
  • Bubble internal portal updated with linked IDs
  • Confirmation emails sent automatically
  • Total: seconds, zero manual intervention

Impact

  • 30–60 min → seconds — entire RFP intake automated
  • Zero duplication — every entity deduplicated before creation
  • Portal stays current — Bubble updated instantly

Daily PDEV Health Digest & Metrics

Active24 nodes (2 workflows)

Daily Scrubbing (PDEV & Projects) + Daily Scrubbing Notifier (PDEV)

Two coordinated workflows run daily. One computes and stores health metrics for every active requirement and project; the other scores, ranks, and posts a prioritized digest to Slack.

Before

  • Someone manually reviews all active requirements each morning
  • They mentally assess urgency — inconsistent across reviewers
  • Summary written up and posted to Slack by hand
  • Some days it doesn't happen — issues slip through
  • No systematic day-to-day tracking of requirement health
  • Trend analysis impossible without historical data points

After

  • Health metrics computed daily for every requirement and project
  • Each record updated with current metrics in Airtable
  • Dated snapshots saved — building trend history over time
  • Requirements scored objectively by urgency, health, and recency
  • Formatted, ranked digest posted to Slack automatically
  • Enables weekly/monthly reporting from real data

Impact

  • Consistent daily visibility — the digest happens every day, no exceptions
  • Objective scoring — urgency is calculated, not guessed
  • Historical trend data — see exactly when a requirement stalled or accelerated
  • ~20–30 min saved daily — eliminates manual review, write-up, and record-keeping
