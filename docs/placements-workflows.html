AI Shortlisting Insights Fetcher

ActiveAI Agent12 nodes

[Gmail] Shortlisting Insights Fetcher

Monitors incoming Gmail messages from clients responding to VA shortlisting emails. A Groq-powered AI agent reads each reply, classifies the client's intent (accept, reject, request more info), extracts key details, and updates the matching Airtable record automatically.

Before

  • Placement coordinators manually check Gmail for client replies
  • Each reply read, interpreted, and hand-entered into Airtable
  • Replies sometimes missed or delayed — candidates left waiting
  • No structured record of why a client accepted or rejected

After

  • Gmail trigger fires on every qualifying reply
  • AI classifies intent: accept, reject, or clarification needed
  • Airtable record updated with structured classification
  • Every reply captured — no candidates left in limbo

Impact

  • ~10 min saved per reply — no manual triage
  • 100% capture rate — every reply gets processed
  • Structured data — accept/reject reasons available for analysis

STP Channel & Notifications

Active20 nodes (3 workflows)

[AT] Create STP Channel + [AT] STP Requests Notifier + [AT] STP Completed Notifier

Three workflows managing the STP (Straight-to-Placements) lifecycle. STPs are VAs sourced by the Talent Acquisitions team directly from job platforms and endorsed to Placements without completing the company's training program. When an STP is created, a dedicated Slack channel is provisioned. Requests trigger notifications via Gmail and Slack. Completion triggers a final status update to the team.

Before

  • Slack channels created manually for each STP — naming inconsistent
  • STP request notifications sent by hand
  • Completions tracked informally — sometimes overlooked

After

  • STP creation auto-provisions a named Slack channel
  • Requests trigger email + Slack notifications
  • Completion status posted to Slack automatically

Impact

  • Consistent Slack channels — every STP gets one automatically
  • No missed notifications — requests and completions always surfaced
  • ~10 min saved per STP lifecycle

Daily Placements Health Digest

Active30 nodes (2 workflows)

[Scheduled] Daily Scrubbing (Placements) + Daily Scrubbing Notifier (Placements)

Two coordinated workflows running daily. The scrubbing workflow computes health metrics for every active placement and writes scores back to Airtable. The notifier scores, ranks, and posts a prioritized digest to Slack — surfacing the placements that need attention.

Before

  • Someone manually reviews all active placements each morning
  • Urgency assessed subjectively — inconsistent across reviewers
  • Summary written and posted to Slack by hand
  • Some days it doesn't happen — issues slip

After

  • Health metrics computed daily for every active placement
  • Each record updated with current scores in Airtable
  • Prioritized digest posted to Slack automatically
  • The digest happens every day — no exceptions

Impact

  • Consistent daily visibility — objective scoring, not subjective review
  • Historical trend data — see exactly when placements stall
  • ~20–30 min saved daily
